# derisk_common

Language-agnostic workflow and knowledge-structure skills. Assumes: nothing.

- [[git-commit-policy]] — when and how to commit completed implementation work.
- [[handover-prompt]] — end completed work with a copy-pasteable handover prompt so the session can be cleared.
- [[test-driven-development]] — TDD workflow: failing test first, make it pass, refactor; then wrap tests around any behaviour that emerged untested.
- [[maintain-okf-bundles]] — maintaining OKF-compatible Markdown bundles with index, log, YAML metadata, and portable links.
